# 应该使用 TypeScript 还是 Flow？

> 原文：<https://javascript.plainenglish.io/typescript-vs-flow-8562f5af32b3?source=collection_archive---------15----------------------->

![](img/df117ac67e978243c2d1db242804a80c.png)

[TypeScript](https://b.remarkabl.org/2NnKxwc) vs [Flow](https://b.remarkabl.org/3rPOIQ8)

> **TL；博士**:选择打字稿。

在为你的 JavaScript 项目评估一个[类型检查器](https://en.wikipedia.org/wiki/Type_system)时，你应该选择[类型脚本](https://b.remarkabl.org/2NnKxwc)还是[流程](https://b.remarkabl.org/3rPOIQ8)？

总体而言，您应该考虑:

*   使用
*   支持
*   社区
*   趋势

# 使用

类型检查器易于安装和使用吗？有据可查吗？它的性能和速度如何？类型定义多吗？易用性与采用率和生产率直接相关。

# 支持

维护人员能够支持项目并解决外部贡献吗？

[流程](https://medium.com/flow-type/clarity-on-flows-direction-and-open-source-engagement-e721a4eb4d8b):

> Flow 几乎完全由脸书的一个 10 人团队开发…该团队的首要任务是满足我们脸书内部客户的需求…遗憾的是，我们没有资源来开发仅由外部开发人员请求的功能。
> 
> Flow 的代码库非常复杂，很难对 Flow 本身做出贡献，我们公司内部的构建系统、性能测试和集成测试使得审查和集成此类拉式请求非常耗时，因此我们可能会关闭与我们的优先级不一致的 pr。同样，当我们知道我们将无法提供及时的支持时，我们将关闭 GitHub 问题。

TypeScript 有大量的贡献者，在 [GitHub](https://github.com/microsoft/TypeScript) 上非常活跃。

# 社区

社区很好地支持类型检查器吗？其他开发者喜欢用吗？你能在[谷歌](https://www.google.com/)或 [Stackoverflow](https://stackoverflow.com/) 上轻松找到答案吗？问题和拉动请求是否得到充分解决？其他开发人员会为类型检查器构建工具、插件和集成吗？一个强大的社区等同于一个稳固的行业地位。

# 趋势

类型检查器的未来是什么样子的？贡献者在发布功能和修复 bug 吗？公司和学校是否会选择其中一种类型的检验员？这可以确定拥有类型检查经验的开发人员的数量。问问你自己，“五年后，哪个类型检查器会赢？”跟随错误的趋势，将会增加[技术债务](https://en.wikipedia.org/wiki/Technical_debt)和过时的技能，这些在以后会成为问题。

# 结论

您应该选择 TypeScript，因为它具有:

*   更好的性能
*   大力支持
*   一个大社区
*   更多类型定义(见[明确类型化](https://github.com/DefinitelyTyped/DefinitelyTyped)
*   更好的 [Visual Studio 代码](https://code.visualstudio.com/)集成
*   更多动力(见[谷歌趋势](https://trends.google.com/trends/explore?date=today%205-y&q=%2Fm%2F0n50hxv,facebook%20flow%20%2B%20flow%20language%20%2B%20flowtype,%2Fm%2F0hjc5m0,%2Fm%2F03yb8hb,%2Fm%2F0h52xr1)和 [npm-stat](https://npm-stat.com/charts.html?package=babel-core&package=typescript&package=flow-bin&from=2015-01-01)

此外，《心流》的创作者[脸书](https://www.facebook.com/)正在将他们的几个大项目如 [Yarn](https://github.com/yarnpkg/yarn) 和 [Jest](https://github.com/facebook/jest) 迁移到打字稿。当心流的创造者离开心流时，这是一个危险信号。
