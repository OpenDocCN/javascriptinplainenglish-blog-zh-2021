在我学习编码的几个月后，我看到一篇文章说所有的开发人员都应该有一个博客。我不认为这适用于我，因为我是编程新手。我能写些什么呢？我对最新技术和最新趋势一无所知。我几乎不知道如何用 JavaScript 写一个 for 循环。

6 个月后，我尝试了一下。我觉得我对 JavaScript 有了很好的理解，并用 React 构建了一些项目。我决定在媒体上发表我的第一篇文章。从那时起，我开始享受写作，现在它是我日常生活的一部分。写文章有很多好处，我希望我能早点开始。

以下是所有自学成才的开发者今天就应该开始写博客的 4 个理由。这也适用于训练营的毕业生、CS 学生和有经验的开发者。但是我将从自学的角度来写作，因为我来自那里。

# 教一些东西是保留知识的最好方法

> “学习的最好方法是教它”

留住知识，看自己是否真正理解一个话题的最好方法之一就是教。你不能教你不知道的东西。如果你不确定该写些什么，选择一些你最近学到的东西，并尝试用书面形式解释。你需要对这个话题做一些研究，并用一种容易理解的方式来解释它。这种方法会巩固你所学的知识，并帮助你记住它。

你也可以把这些文章当成自己以后的小抄。学习编码不是为了记忆东西；它是关于学习如何解决问题。为未来的自己准备一份备忘单是个好主意。

我选择写的第一个主题是数据结构。作为一个自学成才的开发人员，这个领域很容易被忽略。我写数据结构的目的是加深对它们的理解。我需要检查我写的每一行代码，以便在文章中解释它。你可以看看我下面的第一篇文章。

# 让你的名字在那里，建立你的品牌

作为一名自学成才的开发人员，获得第一份工作可能会很艰难！相信我，我正在经历磨难。写文章可以帮助你向潜在雇主推销自己，让你与众不同。你可以向人们展示你最近在使用什么技术，展示你的技能。

这不能保证你得到一份工作，但却是你的第一步。谁知道谁可能会看你的下一篇文章，它可能会带来什么样的机会？

写文章也是建立个人品牌的好方法。看看 Chris Sean 的这段视频，它讲述了作为一名网站开发者，你应该如何建立自己的品牌。

# 帮助其他正在学习的人

学习编码是一个极具挑战性的旅程。你每天都会遇到问题，并且不断地在谷歌上搜索答案。每当我遇到问题时，很多解决方案都不是来自官方文档。答案来自其他开发者的文章或关于堆栈溢出的回答。我数不清 Medium 上的一篇文章或某人的博客帮助我解决了一个问题的次数。

每个人都有一些他们可以从自己的经历中写下或教授的东西。即使你刚刚开始学习编程。我确信你所面临的任何问题，其他人已经或将会遇到同样的问题。如果你为它写了一个解决方案，它可能在将来对某人有益，并且在他们的旅程中帮助他们。

# 创造另一个收入来源

如果你刚刚起步，你可能无法辞去工作成为一名全职作家。但是，如果你投入时间和精力，这是完全可能的。

写文章或开博客可能是给自己开一份被动收入的好方法。我的建议是从这里开始，中等。很容易设置并立即开始写作。他们有一个[合作伙伴计划](https://medium.com/earn)，可以让你赚钱。对于更有野心的类型，你也可以创建自己的博客并在上面投放广告。

我开始写作不是为了挣钱，但每个月看到你的银行账户里有一些额外的收入总是很好的。

# 结论

你可以写的话题数不胜数。如果你遇到了一个问题并找到了解决方案，写下来。如果你刚刚学到了新的东西，写下来。如果你创建了一个有趣的兼职项目，写一个关于它的教程。如果你仍然不确定该写些什么，看看这篇文章[这里](https://medium.com/better-programming/20-types-of-programming-posts-552be6e7d326)获得更多灵感。

感谢阅读！学习编码是一生的旅程，我希望这篇文章已经激励你开始写作。
