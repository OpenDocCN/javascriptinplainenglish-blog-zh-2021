# 介绍

我在 Twitter 做前端软件工程师已经三年了。在旧金山工作之前，我曾在加拿大温哥华的初创公司工作，并为硅谷的大型科技公司的面试做准备。

面试是困难的，当你从另一个国家面试时，这更具挑战性。作为一名候选人和面试官，我想强调一下如何准备前端面试以最大限度地利用你的时间的重要提示。我有两个独立的博客帖子，分别是关于持 TN 签证从加拿大移居美国的[和](https://medium.com/@leannezhang/relocating-to-us-from-canada-with-a-tn-visa-4fb47a1aed34)[我为什么搬家，以及我在加拿大工作期间如何在硅谷找到一份软件工程师的工作](https://leannezhang.medium.com/front-end-interview-preparation-cf186ac7ae55)。在这篇文章中，我想重点谈谈如何准备大型科技公司的前端软件工程面试。

*免责声明:这篇文章并不代表大型科技公司提出的所有问题。它的作用是给出一个概述，让你在面试前集中精力于最重要的部分。*

# 谷歌和亚马逊等大型科技公司的面试结构

通常，他们有三个阶段。

*   第一阶段是最初的技术电话筛选，通常持续 45 分钟。
*   第二阶段是带回家的作业。这可能是可选的，取决于公司。
*   第三阶段是现场面试，或者由于疫情，现在是虚拟面试。

# 第一阶段。初始技术电话屏幕(45 分钟)

1.  介绍— 1 分钟
2.  职业抱负— 5 分钟
3.  通过在线编辑器编码，如 [CoderPad](https://coderpad.io/) — 30 分钟
4.  向面试官提问— 5 分钟

## 面试官如何评价你的技术面试？

*   你是否在很少或没有提示的情况下按时解决了问题？
*   你在编码前花时间去理解和澄清问题了吗？
*   你在面试中大声说出你的想法了吗？
*   你有没有在面试官抓到 bug 之前发现它们？
*   你能让代码更有效率吗？

*准备:请在面试前花点时间熟悉一下*[*coder pad*](https://coderpad.io/)*。*

如果面试官对候选人没有 100%的把握，他们可能会要求候选人在带他们到现场之前再进行一轮电话面试。

# 第二阶段。带回家作业(可选)

这在谷歌、脸书和亚马逊等大型科技公司中并不常见，但在 medium 或网飞等中小型公司中更为常见。例如，他们可能会要求您构建一个简单的 web 应用程序，如拼字游戏或日历，并给你一周的时间来完成它。

## 面试官对此如何评价？

*   你的作业没有出错吗？
*   它捕获了所有的需求吗？
*   它涵盖了所有的边缘情况吗？
*   没有剩余的控制台日志
*   代码不会过度重复
*   代码和注释都很清楚
*   新的函数被很好地命名
*   测试存在吗？如果是，它们有意义吗？

# 第三阶段。现场面试/虚拟面试

现场面试通常需要一整天的时间，主要针对四个方面。

1.  数据结构和算法
2.  前端域
3.  Web 架构和设计
4.  行为面试

## **1。数据结构&算法**

较小的初创公司往往更关注前端领域的知识，但许多大型科技公司仍然重视数据结构和算法，以评估你解决问题的能力。

准备算法的一个好地方是 [HackerRank 面试准备工具包](https://www.hackerrank.com/interview/interview-preparation-kit)或者浏览[破解代码面试](https://www.amazon.com/Cracking-Coding-Interview-6th-Edition/dp/0984782850)的书。因为你是在面试一个前端职位，所以确保你练习了用 JavaScript 解决问题，并且知道大时间的复杂性。如果你对有解决方案的分步面试指南感兴趣，并且不介意付费，可以查看 [educative.io](https://www.educative.io/) 中的[*Ace JavaScript 编码面试*](https://www.educative.io/path/ace-javascript-coding-interview) 。

一旦你熟悉了 HackerRank 面试准备工具包，我建议你在 [LeetCode](https://leetcode.com/problemset/all/) 上练习，从容易到难的问题开始。

总的来说:

*   如果你在找实习，先练习简单的问题。
*   如果你是初级和中级软件工程师，练习简单题和中等题。
*   如果你是高年级学生，做容易、中等和困难的问题。

## 面试官对此如何评价？

*   你在规定的时间内解决问题了吗？
*   你的代码高效吗？
*   你大声说出你的想法了吗？

*提示:由于您很可能会在面试过程中在虚拟白板或纯文本编辑器上写下您的解决方案，因此在练习时，使用钢笔写下您的解决方案，将代码复制到 IDE，并检查它们是否编译成功。*

## **2。前端领域问题**

面试官可能会要求你从头开始构建一个 UI 小部件。最好的准备方式是使用你最喜欢的 IDE 或者网上的 [CodeSandbox](https://codesandbox.io/) 用普通的 Javascript、Html 和 CSS 来编码和构建一个小部件。

**例题:**

1.  设计一个 5 星评级部件
2.  构建一个 7 天天气预报 web 应用程序
3.  制作月历

***温馨提示:***

*   试着学习如何使用原生的 [*获取*](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Fetch) *，这样可以节省你一些时间。*
*   *知道如何使用*[*flexbox*](https://css-tricks.com/snippets/css/a-guide-to-flexbox/)*或*[*CSS grid*](https://css-tricks.com/snippets/css/complete-guide-grid/)*来布局你的 UI 组件。面试官欣赏应聘者对细节的关注。*
*   *理解* [*承诺*](https://www.youtube.com/watch?v=2d7s3spWAzo) *对战回调。从零开始实现承诺，以及如何从承诺* [*到处理错误*](https://www.youtube.com/watch?v=f8IgdnYIwOU) *。背一个新的承诺((resolve，reject) = > {})，这样面试的时候省了不少时间。*
*   *理解* [*闭包*](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Closures)*——能够解释什么是闭包。*
*   *不用看*[*https://javascript30.com/*](https://javascript30.com/)的答案，可以自己动手练习

我最喜欢的学习 JavaScript 的资源:

*   电子书:[雄辩的 JavaScript](http://eloquentjavascript.net/)
*   网址:[学习 JavaScript | MDN](https://web.archive.org/web/20121230133519/https:/developer.mozilla.org/en-US/learn/javascript)

## 3.Web 架构和设计

在回答设计问题时，答案没有对错之分。面试官寻找的是你对网络应用系统的知识深度，以此来决定你能成为什么水平的软件工程师。

准备好回答互联网在高层次上是如何工作的。

*   开放式问题的示例:“从您输入 URL 并按回车键到网站显示在浏览器中，发生了什么？”

**前端设计题示例:**

*   设计一个酒店预订系统
*   向我介绍一下实时聊天应用程序的前端架构

**面试官在找:**

*   有没有明确要求，缩小范围？
*   你能设计出多好的 API？
*   你能把数据库设计得多好？
*   你是如何处理 web 应用程序的 UI 组件和状态的？

我最喜欢的学习 web 架构的资源是来自 Educative 的 [Web 应用&软件架构。](https://www.educative.io/module/web-application-architecture-101)

延伸阅读:[系统设计初级读本](https://github.com/donnemartin/system-design-primer)也很不错，但我发现它对全栈或后端角色更有用。

## 4.行为面试

面试的最后一部分是和你的招聘经理一起进行的。他或她想看看你的简历，听听你的经历和项目。

请突出你的成就，你学到了什么，你想提高什么，你想在你的职业生涯中完成什么。

此外，请准备好要问招聘经理的问题。招聘经理寻找的是你的激情、自我意识和团队配合。
