git lab+NetlifyCMS+Gatsby+Starter =免费、简单、强大

JAMstack是一种现代的、轻量级的方法，用于构建静态内容或者内容随时间变化相对较少的网站。关于你为什么想用JAMstack建立一个网站的其他原因，请看这里列出的的优势。

J 代表一个 JavaScript 前端， A 代表API， M 代表标记。前端通常使用静态站点生成器构建，如这里列出的。后端通常由一个版本控制系统+无头CMS组成，就像这里列出的中的一个。

JAMstack组合的选择

我通常需要一个小型私人网站项目的设置，有以下要求:

  • 免费托管
  • 支持自定义域
  • 内容没有变化或变化很少
  • 没有技术背景的人应该能够自己进行改变

我想出了后端部分的git lab+NetlifyCMS和前端部分的 Gatby 作为我私人项目的完美设置。为了将后端+前端集成为一个功能完整的内容管理系统，我将使用Gatsby-starter-netlify-CMS。这种设置也适用于早期创业。

启用网络CMS

启用Netlify CMS所需的步骤可以在这里和这里找到。在 Netlify 中，在登录页面上选择您的站点。

进入现场设置选项卡，选择标识部分，按下启用标识按钮。

按下身份/注册表部分的编辑设置。

选择仅邀请并按下保存按钮。

转到身份/服务部分并按下启用Git网关。

弹出窗口打开，登录到您的 GitLab 账户。

创建分支

建议不要根据您网站的需求修改模板，而是创建一个网站特定的分支。

  • 转到GitLab repo，通过GitLab UI从master或tag创建分支
  • 转到netlify，站点设置选项卡，构建和部署部分/连续部署/部署上下文，将分支更改为新创建的分支

接下来去哪里？

通常接下来的步骤是

  • 改变站点配置选项，
  • 自定义NetlifyCMS管理仪表板(创建自定义预览，创建自定义小部件)或
  • 自定义正在使用的前端模板。
