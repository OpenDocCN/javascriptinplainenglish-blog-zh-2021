## 即使你不喜欢。

![](img/d5a519a67b0c7bd323a69bf0de0cc346.png)

Photo by [Chris Montgomery](https://unsplash.com/@cwmonty?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

你是自由职业者。你在自己舒适的家里工作。你是你的上司或者老板。“普通人”全职工作的问题似乎与你毫不相干。你没有一个对你的工作成果负责的经理。你不必坐在一个开放的办公室里。你不会在会议上浪费时间。

然而，这并不总是意味着执行订单和保持最高生产率的理想条件。在这篇文章中，让我们回顾一下是什么降低了自由职业者的生产力。

# **1。休息、干扰、会议**

是的，对于作为自由职业者远程工作的程序员来说，这一点是不可忽视的。你不需要和你的主管一起参加团队会议或周会，而是有几个或十几个客户需要关注。

视频会议、电话和电子邮件从标题中喊出“紧急”，无助于保持注意力和专注于编写代码，更不用说解决客户的编程问题了。

几乎没有一个开发人员会“暂停”寻找问题的解决方案，在 Skype 上与一个有时间并想查看订单进展情况的客户聊天，然后“挂断”并直接跳到他不得不停止工作的地方。抓住中断的线索并再次集中注意力有时需要半个小时，有时更长。

干扰越多，挫折、错误和低质量代码就越多。

# **2。微观管理**

微观管理在实践中意味着控制那些并不总是理解你工作方式，而是寻求增加休息、会议、电话和电子邮件次数的人。

分析和判断你所做的每一件事也不愉快地与不信任和削弱你的能力联系在一起。这不仅会影响你的工作效率，还会影响你的工作方式和满意度。

# **3。L** 缺乏精确清晰的指令

缺乏精确和清晰的指令是每个程序员的克星。从诸如“它在这里不工作，修复它”的信息到你将要写的不清楚的规范。在这种情况下，您有两个选择:在客户明确指定之前实现他们的想法，或者等待具体的指示并冒着延迟项目的风险。这一类别还包括优先级不明确的情况，因此您想知道在编码时您是否真的在从事正确的任务。

# **4。简报的意外变化**

根据墨菲定律，一个任务在快完成的时候会被改变，把相对容易的东西变成真正的怪物。

一个简单的功能，例如向网站添加地图，可以发展成一个复杂的 3D 地图制作任务。当这个版本几乎准备好了，客户会询问导航的可能性，最好是一个可爱的动画人。

当然，客户通常没有意识到对他来说一个小的改变对你来说意味着大量的工作，这并没有改变沮丧的程度。

# **5。编码无用的东西**

当然，只要客户付钱，你总是可以告诉自己，你不是顾问或顾问，所以你会写客户想要的任何东西——他的时间，他的钱，他会得到符合准则的最终产品。

更糟糕的是，如果你在工作中途发现这个项目根本不需要，客户开始争论是否要为他们不会使用的东西付费。工作满意度？接近 0 级的地方。

# **6。糟糕的技术解决方案**

即使你有时受够了 bug 和坏掉的代码，但你实际上很享受你的工作，它给了你很大的满足感(当然是在一切正常的情况下)。如果客户为了节省或缩短您的工作时间(从而降低您的成本)，坚持使用质量较低但更便宜、更快捷的解决方案，该怎么办？

富有成效并不总是意味着尽可能快地向前推进并交付解决方案。相反，快速编写的不是最好的代码意味着项目规模的增长和更难管理——有时对您，有时对您的继任者。

# **7。文件质量差**

这就把我们带到了下一点，尤其是在那些经常完成或发展前辈作品的自由职业者中。

每个处于学习阶段的程序员都知道，在代码上留下太多的注释要比太少的好，事实上，每一步的链接都节省了许多开发人员的工作。

如果评论解释了文章做了什么，但没有解释为什么或为什么要做，那就更糟了。效果？Bug 碰到程序里一个你根本不知道是干什么用的片段，你真的不知道从何下手。

# **8。外部工具**

开发人员不怕使用工具进行编程、订购、调试等等。当然，自动化程度越高越好。反过来，如果你使用非常过时的工具，这也会影响你的生产力。

传统键盘和屏幕等“外部”工具也是如此。然而，考虑到硬件成本和程序员的平均工资，即使只提高几个百分点的生产率也是值得投资的。

# **9。我们的工作环境**

在家工作时，你对工作环境有意见吗？这完全取决于你的邻居、室友和/或耳机的质量。例如，在咖啡馆、图书馆或共同工作空间工作并不适合所有的开发人员。你周围太多的运动会分散注意力，但是白噪音或自然的声音——下雨声或海浪声——可以大大提高你的注意力。

# 10。不可能的截止日期

当然，客户希望一切都尽快完成，即使你已经一个星期没有要求一份详细的简报或说明了。那些理解你也需要时间来完成工作的人是最有价值的。

截止日期，这是可行性的边缘。对某些人来说，这可能很刺激。对其他人来说，这可能是不利于思考的沮丧和压力的来源。

如果你仔细观察，我列出的 10 点可以适用于自由职业者在特定项目中从事特定任务的任何行业。然而，可以说他们造成的压力对程序员有更大的影响，因为他们的工作需要大量的注意力和集中来推动进程。
