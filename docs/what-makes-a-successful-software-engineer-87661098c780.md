## 成功软件工程师的 7 个策略

![](img/a81af299ccd16d3643a3cc61cc0b2a43.png)

Photo by [Christina @ wocintechchat.com](https://unsplash.com/@wocintechchat?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

要成为一名成功的软件工程师，你只需要精通你的编码技能；如果你是这样想的，我告诉你，你错了。

我在这里要做的第一件事就是成为一名成功的软件工程师，不仅仅是一名编码软件工程师，而且不仅仅是编码工程师。是关于某种习惯或者策略。

> “我不是一个伟大的程序员；我只是一个习惯很棒的好程序员。”肯特·贝克

当我开始在这个领域工作时，我有一种极端的冲动去开始编码。就在我接到这个项目的时候，一个问题出现了。因为我在下面找到了这些，今天我将与你分享。所以让我们开始吧-

# 1.编码前理解

成功的软件工程师首先理解问题所在，只有这样他们才会完全理解是否应该开始编码。

> 读代码比写代码更难。
> ― **乔尔·斯波尔斯基**

因为理解代码真的很难，一旦理解了代码，就可以自己写了。

假设你有 3o 分钟来解决一个问题；你应该有 25 分钟思考你的问题，5 分钟写代码。

# 2.证明文件

![](img/d93a982ac6274a43dd826bca83a107f8.png)

Photo by [Ben Mullins](https://unsplash.com/@benmullins?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

软件工程师真的很擅长文档。也许你不是唯一一个参与这个项目的人。其他工程师、设计师、数据科学家或产品经理也可能参与其中。

而且你必须确保你脑子里的东西都被记录下来，然后清晰地传达给别人。一个成功的软件工程师会确保记录下他对如何解决问题的想法，问题是什么，可能的障碍是什么。

他们将与所有利益相关者沟通，并参与到这个项目中。他们会这样做，以便每个人都保持在同一页面上，他们也会这样做，以获得每个人的反馈。

也许你忽略了一些东西，你从其他人那里得到的回应是至关重要的。获得响应的最简单方法是在同一页面上为每个人获取该部分的文档。

# 3.编写可读和可维护的代码

> *“任何傻瓜都能写出计算机能理解的代码。优秀的程序员会写出人类能理解的代码。”—马丁·福勒*

成功的软件工程师编写可读的代码。他们这样做是因为他们明白，他们可能不是唯一一个与他们写的东西互动的人，或者可能是他们写的。会有其他人来维护它。

所以他们记住了这样一个事实，必须有人进来看看他们写的这个东西。他们从来没有见过，也不理解。最重要的是，你的解决方案你的代码必须是可读的，因为其他人可能会进来并想要理解它。

# 4.擅长跨职能工作

作为一名软件工程师，你可能会与其他职能部门一起工作，如设计部门或产品经理或数据科学家或运营部门的人员，你称之为成功的软件工程师。他们同情其他有效的活动，几乎把自己放在人的选择中。

这样他们可以有效地与他们交流，他们可以传达他们需要的信息。所以如果我和设计师一起工作，我会试着从他们的设计角度解释我是如何创作的。如果我在与他们交谈时专注于产品的面对面功能，那才是技术实现的真正人工性。

如果我与数据科学家一起工作，我需要就日志游戏或数据分析与他们进行有效的沟通，或者提出正确的问题，说明我们希望如何分析来自某个功能的数据，从而创建新的项目或功能。

所以作为一名软件工程师，擅长跨职能工作真的很重要。

# 5.做一个有效的沟通者

![](img/e9fe845c48095e6532f895055b48d0e9.png)

Photo by [Christina @ wocintechchat.com](https://unsplash.com/@wocintechchat?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

如果你认为作为一个软件工程师，你只是坐在一个角落里却根本不干涉任何人，那你肯定是犯了一个错误；你必须设定自己的界限和沟通方式，但是成功的软件工程师是有效的沟通者。

成功的软件工程师确保他们清晰有效地交流。尤其是当你在一个涉及其他利益相关者的项目中工作，或者你与团队中的其他软件工程师一起工作的时候。你必须确保通过接触与他们沟通。

因为当你交流或进行额外接触时，你要确保每个人都在同一页上，确保每个人都清楚你在做什么，你要确保没有人感到惊讶。

# 6.认识到改进和影响的机会

识别改进和影响的机会，我的意思是，成功的软件工程师不仅仅关注他们正在工作的项目或功能。

但是他们也能够看到坏的一面，因为他们能够跟随他们的队友在做什么。兽人的焦点是什么，为什么它很重要，因为当你看到大画面，而不是太狭隘的权利。

这是我正在做的事情，也是我唯一关心的事情，你能够发现潜在的影响和改进机会。

问问人们为什么以某种方式做事，这样你就能学到与你相关的新东西。你或许可以找出他们系统中的关键文件夹，并加以改进。与人们谈论人们要求你考虑他们如何适应你的工作，这是识别你的团队正在做的事情的机会的一个很好的例子。

# 7.有一个成长的心态

![](img/30b0fceb40a18a02cc09e8b824133515.png)

Photo by [Ravi Roshan](https://unsplash.com/@ravi_roshan_inc?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

> "不管你的职业抱负是什么，学习编码都是有用的."
> *——阿里安娜·赫芬顿，《赫芬顿邮报》创始人*

一定要学会不断地问，一定要有所回应。从错误中学习和反思是成长心态的一个很好的例子。成长心态是认为你的技能是绝对固定的，你可以学习并增加和提高技能，而不是思考。
天赋是有定义的，你的成长是由你付出的努力来定义的。

除了成为一名成功的软件工程师，你还必须有一个成长的心态，才能在任何职业活动中取得成功。你的技能不是一成不变的；你需要不断提高和学习，你绝对需要拥有和认可它们。

成功的软件工程师都在不断学习，不断寻求反馈。事实上，反馈是一件很重要的事情，如果有人没有回应你，他或她不会对此有所防备。

这意味着他们真的很在乎证明你，这样你就能敞开心扉去理解。并不断确定自己可以做得更好的地方。成功的软件工程师对不断发展的学习持开放态度，意识到可能会犯错误。

# 结论

如果你遵循这个策略，它是好的，你的状态也很好。如果你不遵循它，我告诉你，它们是你保持它的惊人策略。
