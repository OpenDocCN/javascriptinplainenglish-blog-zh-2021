## *大节点 js 包列表*

NPM 现在充满了从小到大，从有用到有趣，从伟大到惊人的包装。这是一个“小”列表，列出了我使用过和看到使用过的所有包，从实用程序库到框架。

花点时间在这个列表中滚动，为一些库添加书签，使用或不使用它们。尽情享受这个列表吧，我希望你能从中学习一些新的库。

*您可以通过* `*ctrl+f*` *在列表中搜索您要搜索的任何库或包类型。我会尽我最大的努力在描述中包含每个库的某种通用类别。*

# 1.白垩

[粉笔](https://www.npmjs.com/package/chalk)是一个终端样式库。它可以用来风格化你在终端中所做的一切，比如记录和显示错误。白垩函数可以是链式的，白垩也支持标记的模板文字语法。

# 2.微观的，具体领域的

Vercel 构建了一个小型微服务框架 [Micro](https://github.com/vercel/micro) ，它提供了创建微服务所需的所有基本功能。它具备异步 HTTP 微服务所需的一切，并支持异步/等待。

# 3.矮小的

创建不仅仅是自动递增的身份证可能是一件麻烦事。 [ShortID](https://www.npmjs.com/package/shortid) 有助于此。但是因为它被否决了，我们将使用 [NanoID](https://github.com/ai/nanoid/) 。它会创建一个小于 UUIDs 且仍然安全的字母数字 ID。库本身可以被缩小和压缩到只有 108 字节(！).那就是*小得离谱*。

# 4.jQuery

其中一个 OG 实用程序库。 [jQuery](https://jquery.com/) 是一个众所周知且广泛使用的库，当不使用诸如 React 这样的库时，它可以很容易地进行 DOM 操作。它仍然可以使用，并在 NPM 境内提供。

# 5.Nodemon

当您在开发一个 Node 应用程序时， [Nodemon](https://www.npmjs.com/package/nodemon) 不能错过。Nodemon 使开发 Node 网站变得更加容易。当它检测到文件中的变化时，它会自动重新启动服务器。这样就不必自己重新启动服务器。

# 6.哈巴狗

[Pug](https://pugjs.org/api/getting-started.html) 以前被称为 Jade，是一种可以与 Express 等框架一起使用的模板语言。它源自 YAML 语法，并具有混合和条件语句等特性。

# 7.溺爱

当你使用任何类型的 API 时，环境变量都是非常重要的。Dotenv 使得安全地存储您可以在代码中使用的环境变量变得容易。

# 8.猫鼬

[mongose](https://mongoosejs.com/)是 MongoDB 数据库的对象建模库。如果您正在查找关于 MongoDB 和 JavaScript 的教程，您很可能会以某种方式偶然发现这个库。

# 9.表达

[Express](https://expressjs.com/) 是一个节点服务器框架，它只提供创建一个好的服务器所必需的功能。由于它的极简主义性质，它有一个巨大的社区，为你可以使用的各种任务创建库，并且相当多的将在这个列表中。

# 10.引导程序

大家都知道[自举](https://getbootstrap.com/)。Bootstrap 是一个样式框架，用于创建具有简单样式和良好整体外观的响应性网站。很多网站使用 Bootstrap，有些用在另一个主题上，有些用在一些组件上。它还为一些组件提供了 JavaScript，可以毫不费力地创建一个好网站。

# 11.GraphQL

[GraphQL](https://graphql.org/) 是一种用于 HTTP 请求的查询语言。它希望添加 REST API 的基本功能，同时增加查询语言可能带来的灵活性。它是由脸书开发的，默认情况下随 GatsbyJS 一起提供，用于检索数据。

# 12.摩根

[Morgan](https://www.npmjs.com/package/morgan) ，以我们在电视上最喜欢的‘杀手’命名，是一个日志中间件，通常与 Express 或类似的节点服务器一起使用。它可以随时记录你想要的任何东西。这在调试生产服务器时可能很有用。

# 13.序列

Sequelize 是一个基于 promise 的 ORM，用于 PostgreSQL、SQLite 和 MariaDB 等关系数据库。这是众所周知的，因此，许多教程使用这个库。

# 14.护照

使用最广泛的认证库之一是 [Passport](http://www.passportjs.org/) 。它是一个认证中间件，广泛用于节点应用程序。它支持各种认证，比如 OAuth 和社交登录。

# 15.提前

[Preact](https://preactjs.com/) 是一个最小的 react 克隆，提供了更小的文件大小和更快的速度。它的语法非常相似，如果你知道 React，你就知道 Preact。Preact 只有 3kb，这使得它在大小很重要的地方很有用。和 React 一样，它也是一个前端和 DOM 操作库。

# 16.洛达什

一个非常流行的实用程序库是 [Lodash](https://lodash.com/) 。它提供了数组、字符串和数字操作的函数，这样您就不必自己编写它们了。这个库在几乎任何项目中都很有用。

# 17.尾翼 CSS

我最喜欢的图书馆。 [TailwindCSS](https://tailwindcss.com/) 是一个用于前端开发的样式库。它提供了大量的实用程序类，可以用来代替 CSS。它提供了在开发过程中最小化生产文件大小的功能以及所有类的灵活性。

# 18.Axios

Axios 是一个著名的基于异步承诺的 HTTP 请求库。它简单易学，容易使用，并且不需要你每次都不想发送或设置的任何东西。我有时使用这个图书馆，我不能想象一个更好的。

# 19.阿多尼斯

像 Express 一样，Adonis 也是一个节点服务器框架。不过，它更侧重于微服务方面。它有你需要的一切，它有一个很棒的网站，有文档和例子。

# 20.领域

创建数据库和连接 ORM 可能很烦人。[境界](https://realm.io/)就是你需要的全部。Realm 类似于数据库工具 Mongoose，但是它也提供数据库。这是一个基于对象的，或 NoSQL 数据库。我在手机上结合 React Native 使用过这个，效果很好。

# 21.盖茨比

[GatsbyJS](https://www.gatsbyjs.com/) 是一个遵循 JamStack 的框架。它是一个静态站点生成器，对于创建营销和博客页面非常有用。它提供了编译后超快的速度，他们甚至有 merch。它是基于 React 的，并具有 GraphQL 来从开箱即用的文件中检索数据。

# 22.国际光子

日期和时间是最令人讨厌的工作对象。这就是为什么你应该使用 Moment 或 Luxon。Luxon 更加现代，使用了`Intl`对象，而不是常规的`Date`对象。语法相似，易于使用和学习。

# 23.骗子

每当你创建一个原型，你需要假数据。造假者提供虚假数据。从名字和姓氏到伪造的比特币地址和电子邮件地址。它易学易用。

# 24.埃斯林特

编写漂亮的代码很难。我们都用制表符或空格，要么两个要么四个，有的在圆括号和中括号之间加空格，有的不加。为了使代码更加统一，你可以使用 [ESLint](https://eslint.org/) 。JavaScript 的短评。

# 25.显著的

我们都(可能)知道降价是惊人的。不用鼠标就能快速创建丰富的文本是上帝赐予的礼物。标记为的[帮助我们解析这个文本，并输出我们实际呈现它所需的 HTML。](https://github.com/markedjs/marked)

# 26.使更多

言语艰难。零蛋。一个鸡蛋。两个鸡蛋。如果你曾经使用动态数据，你可能需要正确的复数。同名的[库](https://www.npmjs.com/package/pluralize)提供了该功能。他们甚至允许自定义复数规则。

# 27.同时地

[并发](https://www.npmjs.com/package/concurrently)允许您彼此并发地运行任务，或者说，几乎同时运行，这取决于您的系统。具体来说，它允许您同时运行多个 NPM 任务，如捆绑或编译 CSS 文件。

# 28.苗条的

在 React、Vue 和其他模板框架中，还有 [Svelte](https://svelte.dev/) 。Svelte 允许开发者创建组件并在你的网页中使用它们。它比其他框架更简洁，但是提供了很好的体验。

# 29.髭

[Mustache](https://mustache.github.io/) 是一种无逻辑的模板语言，适用于需要少量动态内容的 Express 应用程序。

# 30.UUID

如果你对 NanoID 不满意，或者你需要真正的 UUID 或 GUIDs， [UUID](https://www.npmjs.com/package/uuid) 就是你要的库。它用最少的代码创建有效的 UUIDs。这些可以在某些数据库中使用，并且比自动增加的 id 更安全。

# 31.HTML-minifier

最终的库是一个微型的。 [HTML-minifier](https://www.npmjs.com/package/html-minifier) 最大限度地减少了网页需要加载的代码量，并且能够*极大地*提高加载速度，从而带来更好的用户体验。您可以为某些最小化样式或规则添加选项，以便该库可以适合每个项目。

# 结论

这甚至不是 NPM 的冰山一角。这个列表中的每个包都是由令人惊叹的开源社区用爱制作的，我要感谢他们的工作。

我希望你已经从这个列表中学到了一两个新的库，并且我希望你欣赏我对这个库的一个小描述和代码片段。
