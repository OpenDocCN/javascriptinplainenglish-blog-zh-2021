# 为什么我不打算在 2021 年购买任何新的编码课程

## 我是如何失去我的技能和热情的。

![](img/53568c2f742c5ced2c6c27b46accc8a4.png)

你知道我为什么讨厌编码教程吗？

因为我被卖给了一个幻觉，这都是我的错，因为那个幻觉是我自己编造的。

我 14 岁就开始编程了。没有人指导我获得成功所需的所有信息。我从一个教程跳到另一个，买了无数的 Udemy 课程。

学完这门课程后，我将准备好成为一名专业人士。在这个大师班之后，我将成为一名高级反应开发者。

你猜怎么着？那从未发生过。

今天，我是一个快乐的专业 web 开发人员，但是让我告诉你为什么我会陷入教程地狱，以及我如何摆脱它。

## 他们卖给我一个开发商的假点子

我永远不会忘记当我看到我的编码老师在我面前的键盘上挥舞双手时，我眼中的惊奇。我一直在想:“这个人怎么能这么快解决这些问题？真是个天才”。

从来没想过我看的视频都是剪辑过的。在发布这些内容之前需要几个小时的研究。分析了问题，进行了头脑风暴。老师一次又一次地尝试各种可能的解决方案，以找到最好的一个。

也许在这个意义上我一直都很天真，但我当时相信我所看到的。直到现在我才认识到我工作的真实性。每天谷歌多少，问同事多少问题。

编码从来都不是一帆风顺的过程。但我花了一段时间才明白。

## 他们让我变得懒惰

我不喜欢离开我的舒适区。我是说，谁知道呢？我喜欢实现我的目标，但是得到好东西的痛苦过程从来都不是愉快的。

练一门自己不懂的技能，可谓是一个痛苦的过程。你感到无知，无法理解。

但这正是编程课程能派上用场的地方。我会遵循一条路径，写下老师提出的所有代码，并立即感到富有成效。不幸的是，真正重要的是你写的代码背后的想法，而这正是将想成为专家的人培养成专家的精神努力。

## 他们让我远离了我唯一要做的事情

从我决定成为一名软件开发人员的那一刻起，我只应该做一件事:构建我的项目。

我应该培养自己的思想，而不是一个课程库。每当我脑子里有了很酷的东西，我就应该努力去理解如何建造它。这就是开发人员的特质:渴望和挑战。

理论是好的，但它的目的只是给你一些基础知识，让你知道你做的事情是否正确。理论的范围是帮助你自我监控自己，但实践是建立技能的。

## 我的外卖钥匙

不要误解我这篇文章的意思。编码教程也帮助了我。他们给了我建立技术技能的基础，他们让我作为专业人员不断更新，我甚至自己作为老师创建了免费的 JavaScript 课程。

我掉进了教程的地狱，都是我的错。我是那个允许编码教程把我从实践我的激情和技能中带走的人。这就是为什么我决定在今年剩下的时间里不再买它们。我想回到青少年时代，只是练习。我只想把我脑子里的东西编码出来。我想再次感觉像个孩子。

我不希望在完成一门新课程后，我的技能会有所提高，但只有一次，我会用我发现的新事物来构建东西。

所有这些对你来说可能听起来很傻，但这是我重新点燃激情的方式。我知道我在学习新的编码技能方面有问题，这就是为什么我需要自己休息一下。

如果你发现自己处于同样的状况，请三思是什么原因造成的。你是否将你的改进委托给某个外部实体？新课程，新的大师课？还是刻意练好自己的技能，和自己的激情一起成长？

—皮耶罗
