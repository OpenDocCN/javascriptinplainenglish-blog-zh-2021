Docker改进了他们的Mac M1支持

几个月前，我和Docker一起开发了我的 Macbook Pro (M1) 。我安装了它，但遗憾的是，我的mac立即崩溃了。

因为我不需要它，所以我把它放在一边。

几个月后，我需要Docker。我装的；安装过程中一切正常。即使在尝试旋转Docker教程容器时，它仍然在运行🥳

如何在Mac M1上使用Docker？

在安装Docker for Mac之前，在你的终端中运行这个命令进行 安装Rosetta 2 。如果你已经安装了它，你可以跳过这一步

softwareupdate --install-rosetta

从Docker网站下载 Docker for Mac 并安装。

什么Docker图像可以在Mac M1上使用？

许多码头工人图像可用于M1。不是每个Docker映像维护人员都这样做的。Docker创建的大多数官方Docker映像都支持ARM64。

带有ARM64标签的Docker图像原生运行在Mac M1上。如果您使用Rosetta 2，您可以运行amd64映像，但它们可能会导致性能问题。

支持ARM64的常见Docker图像示例如下:

  • Ubuntu
  • Python
  • Redis
  • Postgres
  • Nginx
  • 节点JS
  • Traefik
  • 蒙哥
  • 马里亚DB
  • OpenJDK
  • 按下
  • 执政官
  • phpMyAdmin
  • 红宝石
  • 还有更多…

如果您发现Docker图像不支持ARM64，请转到Github问题页面并打开一个问题来请求ARM64支持，这将帮助所有人👍

MSSQL

对于每个需要在应用程序中使用MSSQL的人来说，我发现这个图像还不支持ARM64。这个Docker图像的替代图像是mcr.microsoft.com/azure-sql-edge:latest。我在不同的调试过程中发现了这一点，但最终让它与这张图片一起工作。
